| Тип теста   | Аннотация                           | Загружает       |
| ----------- | ----------------------------------- | --------------- |
| Юнит        | @ExtendWith(MockitoExtension.class) | Ничего          |
| Интеграция  | @SpringBootTest                     | Полный контекст |
| Контроллеры | @WebMvcTest                         | Только MVC слой |
| Репозитории | @DataJpaTest                        | Только JPA      |

Пример:
@SpringBootTest(
classes = MyApp.class,        // Загрузить конкретный конфиг
webEnvironment = MOCK,        // MOCK/REAL/RANDOM_PORT
properties = "spring.datasource.url=h2://test"  // Переопределить properties
)

Что делает
- Автоматически ищет главную конфигурацию (@SpringBootApplication)
- Инициализирует все бины, БД-конфигурации, properties
- Поднимает реальный Spring контекст как при запуске приложения


## Unit тесты

`@ParameterizedTest`
Заменяет обычный @Test — делает метод параметризованным. JUnit вызывает один метод много раз с разными аргументами из источника данных.

`@CsvSource`
Источник данных в формате CSV. Каждая строка = один запуск теста. Колонки → параметры метода.

`@DisplayName`
Название теста в отчёте (вместо имени метода). Человекочитаемо.

`@Nested`
Группирует тесты логически — создаёт подклассы в отчёте. Визуально разделяет позитив/негатив сценарии.
